# Padel Coaching Tech Research

> An evidence-graded research portfolio on padel coaching technology, plus the multi-agent pipeline that produced it. Eight static HTML pages, every claim cited, every page audited. Apache 2.0.

[![License: Apache 2.0](https://img.shields.io/badge/license-Apache_2.0-blue.svg)](LICENSE)
[![Live site](https://img.shields.io/badge/live-avaluev.github.io-success)](https://avaluev.github.io/padel-market-analysis/)

## What this is

Two things in one repository:

1. **A public research portfolio.** Eight HTML pages on padel coaching technology in 2026 — competitor landscape, subscription economics, MVP design, ninety-day operating plan, and full disclosure of every AI model used. Live at <https://avaluev.github.io/padel-market-analysis/>.
2. **The pipeline that produced it.** Markdown sources in `reports/sources/deliverables/`, deterministic builders in `scripts/`, content quality gates in `scripts/check_quality.py`, AI-search-optimisation assets generated by `scripts/build_seo_assets.py`, and the agent definitions in `.claude/agents/`.

The pipeline ships as static HTML with no build step. The deploy mechanism is GitHub Actions watching `reports/final/`.

## Quick start (rebuild the site)

```bash
make install      # python dev deps + node deps
make build        # rebuild all pages + SEO assets
make audit        # run all quality gates
```

Or step by step:

```bash
node scripts/build_pages.mjs          # markdown → HTML pages
node scripts/inject_topnav.mjs        # inject the standard nav
python3 scripts/build_seo_assets.py   # robots.txt, llms.txt, sitemap.xml, etc.
python3 scripts/check_quality.py      # run quality gates
```

## Quality gates

The `scripts/check_quality.py` gate enforces every promise the project makes about its public pages:

| Gate | What it blocks |
|---|---|
| `h1` | Multiple `<h1>` per page |
| `run_id` | Internal run-ID timestamps in published prose |
| `candidate` | "candidate" framing leaked into prose |
| `jd_coverage` | Reference to the deleted JD coverage page |
| `jargon` | Configurable jargon blocklist (kill, north star, red team, etc.) |
| `marketing` | Specific unverified marketing badge phrases |
| `link_syntax` | Literal `<https://...>` markdown auto-link text |
| `internal_ids` | Internal codes (`VM-001`, `CH-006`, etc.) in prose |
| `meta` | Required meta tags missing (title, canonical, OG, Twitter) |
| `jsonld` | JSON-LD structured data missing or invalid |
| `images` | `<img>` without alt + width + height |
| `links` | Broken internal links |
| `nav` | Inconsistent navigation across pages |
| `seo_assets` | Missing site-wide SEO files |

Each gate has unit tests under `tests/`. The gate runs on every pull request via `.github/workflows/quality-gates.yml` and blocks merge to `main` on failure.

## AI search optimisation

The site is engineered for citation by ChatGPT Search, Perplexity, Gemini, Claude, and Google AI Overviews. The four paradigms (GEO, AIO, AEO, LLMO) are documented in detail at `evidence/research/ai-search-optimization/SUMMARY.md`. Every public page has:

- A 40–60 word citable summary lead under H1.
- JSON-LD `Article` + `Organization` + `Person` + `BreadcrumbList`.
- Open Graph + Twitter Card meta tags.
- Canonical URL.
- Mobile-first navigation with hamburger drawer.
- Server-side rendered HTML (no JS framework).

Site-wide:
- `robots.txt` allows GPTBot, OAI-SearchBot, ChatGPT-User, ClaudeBot, Claude-SearchBot, Claude-User, PerplexityBot, Google-Extended, Applebot-Extended.
- `llms.txt` and `llms-full.txt` per the llmstxt.org spec.
- `sitemap.xml` with `lastmod` from file mtime.

## Tooling

The pipeline uses Python (stdlib only at runtime; `ruff` / `mypy` / `pytest` / `pre-commit` for development), Node 20 with Playwright for mobile auditing, and `bash` for orchestration.

OpenRouter is the inference plane. `OPENROUTER_API_KEY` must be exported before any pipeline run. The compute allocation across models is documented in `CLAUDE.md` and `QUALITY_BAR.md`.

## Reproducing a research run

```bash
export OPENROUTER_API_KEY=...
bash scripts/run_pipeline.sh
```

This runs the full sequence (sixteen builder scripts, parallel research arms, gate checks, mobile audit). See `RUN.md` for the phase-by-phase reference.

## Contributing

See `CONTRIBUTING.md` for the change workflow and how to add a new gate. Pull requests must pass:

- `make lint` (ruff)
- `make typecheck` (mypy --strict)
- `make test` (pytest)
- `make check-quality` (content + SEO gates)
- `make check-links` (HTTP 200 on every URL)

The `.pre-commit-config.yaml` runs the same checks on every commit.
