# Contributing

Thanks for taking time to look. This is a single-author research portfolio plus pipeline; contributions are welcome but the bar for what gets merged is high.

## Quick principles

- **Evidence over polish.** Every numeric claim in the published pages cites a source URL. Pull requests that add unsourced numbers will be rejected.
- **Plain English.** Jargon goes through the `check_jargon` gate. If you find yourself wanting to use "kill experiment", "north star", "red team", or any internal acronym in user-facing prose, the gate will block it.
- **No marketing claims.** Specific banned phrases are listed in `scripts/check_quality.py → MARKETING_PHRASES`. Don't add new ones.
- **Tests first.** New gates require unit tests. New content transforms require unit tests.

## Local setup

```bash
make install
pre-commit install
```

## Change workflow

1. Open an issue describing what you're proposing.
2. Branch from `main`.
3. Make the change.
4. Run `make audit` locally — must exit 0.
5. Open a pull request.
6. CI runs the same checks plus `gitleaks` for secrets.
7. The PR is reviewed against the rules in `QUALITY_BAR.md`.

## Adding a quality gate

1. Add a function `check_<name>(path: Path) -> Iterable[Violation]` to `scripts/check_quality.py`.
2. Register it in `PER_PAGE_CHECKS` or `SITE_WIDE_CHECKS`.
3. Add unit tests to `tests/test_check_quality.py` covering both the clean and the violation cases.
4. Run `make test` — coverage must stay ≥ 70% on the gate modules.
5. Document the gate in the README's quality-gate table.

When a new content failure is found in production, that failure becomes a new gate. The rule is: no retry without a new gate.

## Adding a builder

1. Source markdown goes in `reports/sources/deliverables/`.
2. Add a registry entry to `scripts/build_pages.mjs → PAGES`.
3. Run `node scripts/build_pages.mjs` and verify the output.
4. Run `make audit` and fix any gate violations.

## Editing existing pages

- For pages generated by `build_pages.mjs` (the five report pages plus the index), edit the source markdown, never the rendered HTML.
- For pages that are hand-edited (`index.html`, `methodology.html`, `padel-ai-coach-research.html`, `evidence-map.html`, `404.html`), edit the HTML directly. Run `python3 scripts/sanitise_pages.py` afterwards to apply the standard cleanups.

## Style

- Python: ruff + mypy --strict (configured in `pyproject.toml`).
- JavaScript / Node: no formatter configured yet; match surrounding style.
- Markdown: H1 once per file; blockquote summaries; `[text](url)` over `<url>` auto-links.
- HTML: semantic tags; alt text on every image; width/height on every image.

## Commit messages

Conventional commits style:

```
fix: correct Y
refactor: simplify Z
docs: update README
test: add coverage for W
chore: bump dep
```
